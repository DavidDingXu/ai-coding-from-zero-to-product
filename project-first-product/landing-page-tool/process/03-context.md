# 03 给 AI 的上下文

## 背景

这是《从想法到产品：普通人的 AI 编程实战》的第一个实战项目。目标不是做复杂应用，而是让读者第一次看到“从想法到可运行页面”的完整过程。

## 需要 AI 遵守的项目规则

- 项目结构要简单。
- 不使用 React、Vue、Vite、Next.js 等框架。
- 不引入第三方依赖。
- 核心生成逻辑要能脱离浏览器单独测试。
- 页面交互代码和生成逻辑分开。
- 所有命令必须能在 README 中找到。
- 完成后必须给出测试和浏览器验证方式。

## 初始文件设想

```text
landing-page-tool/
├── README.md
├── index.html
├── styles.css
├── package.json
├── src/
│   ├── app.js
│   └── brief-builder.js
└── test/
    └── brief-builder.test.js
```

## 技术选择

选择原生 Web 技术，不是因为它最强，而是因为第一讲要降低干扰。

如果一开始就引入框架，读者会把注意力放在脚手架、依赖安装、构建命令和框架概念上。这个项目只需要讲清楚三件事：

- 表单如何收集输入。
- 生成逻辑如何输出简报。
- 测试和浏览器如何验证结果。

## 给 AI 的重点提醒

- 不要把最终源码当成唯一交付。
- 每一步都要能解释为什么这么做。
- 不要一次性做完复杂功能。
- 优先保证刚开始也能跑起来。
