# 02 需求整理

## 产品名称

AI 产品简报生成器

## 用户故事

作为一个 AI 编程初学者，我希望把一句模糊的产品想法整理成结构化简报，这样我可以把简报交给 Codex App 继续开发，而不是只让 AI 猜我的需求。

## 输入

| 字段 | 是否必填 | 说明 | 示例 |
|---|---|---|---|
| 产品想法 | 是 | 一句话描述想做什么 | 做一个给上班族用的个人效率计划工具 |
| 目标用户 | 否 | 谁会使用这个工具 | 上班族、产品经理 |
| 核心功能 | 否 | 第一版必须具备的能力 | 记录任务和日程，按优先级排序 |
| 体验风格 | 否 | 页面和交互倾向 | 清爽，适合长时间使用 |
| 限制条件 | 否 | 第一版不做什么 | 先做本地单页版本，不接后端 |

## 输出

生成两段内容：

1. 产品简报。
2. 给 AI 编程工具的提示词。

产品简报必须包含：

- 产品名称
- 一句话想法
- 目标用户
- 核心功能
- 体验风格
- 限制条件

给 AI 编程工具的提示词必须要求：

- 先澄清需求。
- 再生成实现计划。
- 优先做最小可用版本。
- 完成后说明如何运行和验证。

## 交互

- 页面左侧显示表单。
- 页面右侧显示生成结果。
- 未生成前，复制按钮不可用。
- 生成后，可以复制提示词。
- 产品想法为空时，页面展示错误提示。

## 工程边界

- 使用原生 HTML、CSS、JavaScript。
- 使用 Node 内置 `node:test` 写测试。
- 不引入打包器。
- 不引入前端框架。
- 不引入第三方依赖。

## 完成判断

这个项目完成，不是因为代码看起来完整，而是因为以下动作都能成立：

- `npm test` 通过。
- `npm start` 能启动本地页面。
- 浏览器能打开 `http://localhost:5173`。
- 填入示例想法后，页面能生成简报和提示词。
- 复制按钮能复制生成后的提示词。
