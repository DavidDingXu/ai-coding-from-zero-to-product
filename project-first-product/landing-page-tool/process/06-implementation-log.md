# 06 实现记录

## 第 1 轮：先写测试

先写 `test/brief-builder.test.js`，覆盖两个行为：

- 正常产品想法能生成产品简报和 AI 编程工具提示词。
- 产品想法为空时抛出错误。

第一次运行测试，预期失败，因为 `src/brief-builder.js` 还不存在。

失败信息是：

```text
ERR_MODULE_NOT_FOUND
```

这个失败是有价值的。它证明测试真的在约束尚未实现的代码，而不是写了一个永远通过的空检查。

## 第 2 轮：实现核心规则

创建 `src/brief-builder.js`，实现：

- `buildBrief(input)`
- 输入清理
- 空想法校验
- 产品名称提取
- 产品简报拼接
- AI 编程工具提示词拼接

之后重新运行测试。

## 第 3 轮：修正标题提取

测试要求：

```text
做一个给上班族用的个人效率计划工具
```

生成标题：

```text
上班族个人效率计划工具
```

初始实现没有处理“给……用的”这个中文表达，标题不够自然。

修正方式是在 `makeTitle` 中增加这一条规则：

```js
.replace(/^给(.+?)用的/, "$1")
```

这不是为了做复杂自然语言处理，只是覆盖第一讲使用的高频表达。

## 第 4 轮：接页面

创建：

- `index.html`
- `styles.css`
- `src/app.js`

页面交互链路：

```text
表单提交 -> FormData 读取输入 -> buildBrief 生成结果 -> renderResult 渲染页面
```

同时处理两种状态：

- 生成成功：展示产品简报和提示词，启用复制按钮。
- 生成失败：展示错误，禁用复制按钮。

## 第 5 轮：浏览器验证

启动本地服务后打开：

```text
http://localhost:5173
```

输入示例产品想法，点击“生成简报”，确认右侧出现：

- 产品名称
- 一句话想法
- 目标用户
- 核心功能
- 体验风格
- 限制条件
- 给 AI 编程工具的提示词

再点击“复制提示词”，确认剪贴板里包含生成内容。

这一步验证的是页面真实行为，不是源码静态存在。
