# 05 plan

1. 先把官网需求写清楚：目标用户、首屏承诺、候补表单字段、不做范围。
2. 写测试，锁定官网项目的范围和表单行为，避免实现时扩成完整应用。
3. 确认红灯：`src/landing-page.js` 不存在，说明测试已经指向核心逻辑。
4. 实现 `landing-page.js`，提供文案、校验、候补记录、检查清单和 AI 提示词。
5. 实现 `index.html`、`styles.css` 和 `src/app.js`，把核心逻辑接到浏览器页面。
6. 实现 `cli.js`，让 README 里的命令能输出验证结果；CLI 只作为辅助验证，不作为实战主体。
7. 补 README 和过程记录，说明当前是本地 demo，不接模型、登录、支付和远程数据库。
8. 运行 `npm test`、`npm run brief`、`npm run feedback`、`npm run checklist`。
9. 启动本地页面，用浏览器检查首屏、空表单、错误邮箱、密钥文本和完整表单。

## 取舍

本轮不引入前端框架。官网第一版只需要静态 HTML、CSS 和少量浏览器脚本。

这样可以直接看懂文件结构，读者也能看到测试、页面和 CLI 之间的对应关系。

本轮不接真实模型和数据库。官网阶段要验证的是产品表达和候补反馈，不是模型质量和账号体系。过早接服务会把 Key、费用、跨域、隐私和部署问题带进来，反而看不清官网需求是否成立。
