# 01 idea

主线产品是「AI 个人效率助手」。

这个案例来自一个真实的早期产品场景：想做个人效率工具的人很多，但一上来就做完整应用，会很快陷入登录、模型、数据库、提醒和支付。第一轮更应该先验证别人能不能看懂产品承诺。

第一版官网要解决的问题不是生成今日计划，而是先把产品价值说清楚，并收集第一批候补用户反馈。

目标读者包括学生、上班族、产品经理、运营人员、独立开发者和正在学习 AI 编程做产品的人。

这一版先让读者看到一个可运行页面，也看到这条需求为什么这样切：

- 首屏说明产品解决什么问题。
- 价值卡片解释为什么有用。
- 三步流程说明后续网页应用会怎么做。
- 候补表单收集称呼、邮箱、角色、领域和痛点。
- 表单只做本地校验，不上传远程服务。

真实约束：

- 不能要求用户填写 API Key、密码、账单或真实日历内容。
- 不能把“后续会接模型”写成“当前已经接模型”。
- 候补表单只能收集判断下一轮需求所需的最小信息。
- 当前只验证官网表达和本地反馈，不验证账号、远程保存、模型质量和提醒能力。

下一轮反馈问题：

- 读者是否能用一句话复述这个产品做什么。
- 候补用户最常选择哪个效率场景。
- 用户描述的痛点是否真的集中在任务、日程、笔记和账目整理。
