# web-landing：AI 个人效率助手官网和候补表单

这是「AI 个人效率助手」产品实战的第一个可运行项目，对应第 18 篇。

这一版只做官网、价值说明、候补表单、本地校验和页面内反馈。它不接真实模型、不保存远程数据库、不做登录支付。

## 为什么先做官网

很多产品一开始就想做完整应用，结果登录、模型、数据库、支付全挤在一起，页面还没跑稳，范围已经失控。

`web-landing` 先解决一个更小的问题：

- 用一句话说明产品解决什么问题。
- 展示第一版流程。
- 收集候补用户的角色、领域和痛点。
- 验证表单边界和隐私提示。
- 为第 19 篇网页应用留下清晰入口。

## 运行方式

进入项目目录：

```bash
cd project-vibe-lab/web-landing
```

运行测试：

```bash
npm test
```

预期结果：

```text
tests 7
pass 7
```

查看产品边界：

```bash
npm run brief
```

验证候补表单数据结构：

```bash
npm run feedback
```

启动本地页面：

```bash
npm start
```

浏览器访问：

```text
http://127.0.0.1:5174
```

## 页面验证

打开页面后，建议做三次检查：

1. 首屏能看懂「AI 个人效率助手」解决的问题。
2. 提交空表单，应看到错误提示。
3. 填写称呼、邮箱、角色、领域和痛点，应看到本地候补记录。

当前 demo 只在浏览器页面展示校验结果，不上传远程服务。

## 目录说明

```text
web-landing/
├── index.html
├── styles.css
├── src/
│   ├── app.js
│   ├── cli.js
│   └── landing-page.js
├── test/
│   └── landing-page.test.js
├── process/
└── .env.example
```

`src/landing-page.js` 放页面文案、表单校验和候补记录构造逻辑。页面和测试都使用它，避免文章里讲一套、代码里做另一套。

## 当前边界

- 不接真实模型。
- 不写远程数据库。
- 不做账号体系。
- 不接支付。
- 不自动发布内容。
- 不要求用户填写 API Key 或密码。

这些能力会在后续项目逐步接入。
