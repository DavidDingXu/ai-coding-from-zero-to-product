# 05 实现计划

1. 先把官网之后的真实需求写清楚：用户输入任务、日程、笔记、账目和今日目标，得到可取舍的行动卡。
2. 写 `test/plan-app.test.js`，覆盖范围、输入默认值、10 张行动卡、筛选、本地保存、提示词和检查清单。
3. 确认红灯，确保测试先指向还不存在的核心逻辑，而不是先写页面。
4. 创建 `src/plan-app.js`，实现独立核心逻辑；页面、CLI 和测试都复用它。
5. 创建 `index.html`、`styles.css` 和 `src/app.js`，实现浏览器页面的输入、生成、筛选、复制、保存和清空。
6. 创建 `src/cli.js`，让读者能用命令验证生成、筛选、保存和检查清单；CLI 只是辅助验证。
7. 补齐 README 和 process 记录，说明当前是本地规则生成和 localStorage 保存。
8. 运行测试和 CLI 命令。
9. 启动本地页面，检查生成、筛选、复制、保存、刷新保留、清空，以及是否有遮挡、重叠或文字不清。

## 取舍

第一版不引入前端框架。原因是本轮目标不是学习 React 或 Next.js，而是证明网页应用主流程。

第一版不接模型。原因是模型接入会带来 Key、费用、接口稳定性和响应解析问题，容易掩盖产品流程自身的问题。

第一版不写远程数据库。原因是保存动作先用 localStorage 就能验证交互和数据结构，后续再替换成后端 API。

第一版也不读取真实日历、聊天记录或账单。原因是这会引入授权、隐私和平台接入问题。本轮先让用户手动输入，验证产品主路径是否成立。
