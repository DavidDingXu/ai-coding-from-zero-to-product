# 08 复盘

这轮做对的地方：

- 先用测试锁住核心逻辑，再做页面。
- 页面、CLI 和测试都调用 `src/plan-app.js`，没有各写一套规则。
- 生成、筛选、保存都能用命令验证。
- 当前版本明确不接真实模型和远程数据库，避免把 demo 能力写过头。

后续要注意：

- 接模型时，不能直接把 API Key 放到前端页面。
- 接后端时，要保留 localStorage 版本作为离线 demo 或降级路径。
- 行动卡质量不能只看标题，要继续补理由、下一步动作和素材需求。
- 如果后续引入 React 或 Next.js，先保证当前测试仍然能跑，不要把核心规则写进组件内部。
