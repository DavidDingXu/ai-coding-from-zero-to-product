# 02 需求整理

## 本轮目标

完成一个可运行的网页应用，支持：

- 输入今天的任务。
- 输入日程和时间限制。
- 输入笔记和零散信息。
- 输入账目或成本提醒。
- 输入今日目标和约束。
- 生成 10 张今日行动卡。
- 按难度、内容类型、目标阶段筛选。
- 复制单张行动卡。
- 保存单张行动卡到浏览器本地。
- 清空本地计划记录。
- 用测试和 CLI 验证核心逻辑。

## 本轮不做

- 真实模型调用。
- OpenAI、DeepSeek、千问或兼容网关接入。
- 登录、注册、团队协作。
- 远程数据库保存。
- 支付、额度、会员。
- 自动发送真实提醒。

## 用户和场景

典型用户：

- 上班族：上午有会议，下午要交材料，还夹杂几条聊天里来的待办。
- 产品经理：有需求评审、用户反馈、文档修改和数据复盘，不知道先排哪几件事。
- 运营人员：要处理活动素材、客户消息、数据记录和预算提醒。
- 独立开发者：要在开发、内容、用户反馈和成本记录之间做取舍。

当前输入不要求接入真实日历、聊天工具或记账软件。用户先手动输入，目的是验证页面主路径和输出结构。

## 数据结构

每张行动卡包含：

```text
id
rank
title
audience
domain
goal
tone
constraints
type
stage
difficulty
angle
score
tags
reason
nextAction
```

保存记录包含：

```text
cardId
title
difficulty
type
stage
score
storage
source
note
savedAt
```

## 验收标准

- 页面能说明当前是本地规则生成，不是模型调用。
- 输入区能承接任务、日程、笔记、账目和今日目标这类真实上下文。
- 生成结果必须有类型、阶段、难度、推荐理由和下一步动作。
- 筛选不能破坏原始 10 张卡。
- 保存记录必须标明 `storage: local-demo`。
- README、文章、页面和测试不能把 localStorage 写成远程数据库。

## 验证方式

- `npm test` 验证核心逻辑。
- `npm run generate` 验证 10 张行动卡输出。
- `npm run filter` 验证筛选。
- `npm run save` 验证本地保存记录结构。
- `npm start` 打开页面，手动验证生成、筛选、复制、保存、刷新保留和清空。
