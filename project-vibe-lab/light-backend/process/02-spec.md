# 02-spec

## 目标

做一个AI 个人效率助手轻量后端 MVP，验证数据保存、登录会话、资源权限、API 合同和审计事件。

真实需求是：当前端本地保存无法承担跨设备、多人访问、密钥保护和写操作追踪时，用一个轻量后端接住最小的数据责任。

## MVP 范围

- 本地内存数据源。
- 模拟登录会话。
- 用户、项目、草稿、审计事件模型。
- 基于角色和项目成员的权限检查。
- 草稿保存和查询。
- API 合同和权限矩阵。
- 生产级升级缺口。

## 不做

- 不接真实数据库。
- 不做真实密码登录、OAuth 或短信登录。
- 不接真实模型 key。
- 不做支付、扣费、发票或真实隐私数据处理。

## 验收标准

- 未登录用户不能保存草稿。
- `viewer` 可以看，不能写。
- `editor` 可以写自己所属项目，不能写不属于自己的项目。
- `owner` 才能管理成员。
- 写操作必须产生审计事件。
- 当前项目必须标明 `productionReady = false`。
- API 合同要先写清 method、path、auth、request、response 和错误场景。

## 生产级升级方向

- 数据库迁移、备份和回滚。
- 真实登录、会话续期和退出登录。
- 基于角色和资源的权限系统。
- 服务端模型代理、额度和限流。
- 请求日志、审计事件、告警和 traceId。
- 数据导出、删除账号、敏感字段脱敏和访问留痕。
