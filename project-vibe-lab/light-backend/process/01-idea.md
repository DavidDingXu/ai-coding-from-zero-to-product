# 01-idea

「AI 个人效率助手」已经有多个前端形态，但数据仍然主要停留在本地演示阶段。

下一步需要回答一个更现实的问题：什么时候必须引入后端？

第一版不做重型后端，不上 Docker，不连真实数据库。先用内存数据模拟用户、项目、草稿和审计事件，让读者看清数据库、登录和权限分别解决什么问题。

这个需求来自真实产品继续迭代时一定会遇到的边界：用户想在手机和电脑之间继续使用，团队成员要看同一份计划，模型 key 不能放在浏览器里，写操作出了问题要能追踪。

本轮不是“加一个后端 demo”，而是先定义后端接手哪些责任：

- 身份：谁在操作。
- 资源：数据属于哪个项目。
- 权限：谁能看，谁能写。
- 写入：草稿如何保存。
- 审计：谁在什么时候做了什么。
- 合同：前端和后端如何约定接口。
