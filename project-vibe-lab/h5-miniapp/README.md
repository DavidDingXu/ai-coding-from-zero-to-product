# h5-miniapp：把网页应用压缩成手机上的短流程

这是「AI 个人效率助手」的第三个实战项目。

前一个 `web-app` 已经能输入任务、日程、笔记和账目，生成 10 张今日行动卡，并支持筛选、复制和本地保存。本项目不继续堆功能，而是做一次更接近真实产品的取舍：把完整网页应用压缩成手机上能完成的 H5 版本。

当前版本不调用真实模型，不做真实小程序上传审核，也不接登录、openid 或云数据库。它交付的是移动端主路径：短输入、3 条推荐、本地收藏、行动文本和发布路径提示。读者可以把同样的方法迁移到自己的产品，先做一个能分享、能验证的移动端版本。

## 先看到手机端短流程

进入项目目录：

```bash
cd project-vibe-lab/h5-miniapp
```

打开本地页面：

```bash
npm start
```

浏览器访问：

```text
http://127.0.0.1:5176
```

页面打开后，先改一次场景、短输入和时间预算，生成 3 条推荐；收藏一条；清空收藏；查看行动文本是否同步变化。

## 再看验证证据

页面主路径跑通后，再用命令确认移动端逻辑、分享文本和发布边界：

```bash
npm test
npm run verify
npm run brief
npm run plan
npm run share
npm run publish
```

`verify` 会同时跑测试、计划/分享命令和页面关键文案检查。

## 当前能做什么

- 把完整网页应用压缩成移动端短输入。
- 根据产品、场景、用户、短输入和时间预算生成 3 条今日推荐。
- 在浏览器 localStorage 里保存收藏记录。
- 生成适合复制到聊天或笔记的行动文本。
- 区分 H5 链接验证和小程序提审准备。
- 用 Node.js 测试验证核心逻辑。

## 当前不做什么

- 不调用真实模型。
- 不上传真实小程序。
- 不接登录、openid、云数据库和消息订阅。
- 不收集手机号、身份证、公司内部资料。
- 不把 H5 本地 demo 写成已经通过平台审核的小程序。

这些能力以后可以接，但第一版先验证移动端主路径。产品还没在手机上跑顺之前，不急着接平台能力。

## 关键文件

`src/h5-miniapp.js` 是核心逻辑。页面和 CLI 都调用它，避免页面能跑但命令不可验证。

`src/app.js` 是浏览器交互。它负责读取表单、渲染推荐、收藏、清空本地收藏和更新行动文本。

`src/cli.js` 给熟悉终端的读者做验证用。你可以在终端直接看边界、计划、推荐、收藏、分享和发布检查项。

`test/h5-miniapp.test.js` 覆盖移动端范围、输入归一化、3 条推荐、本地收藏、分享文本、发布路径和给 AI 的提示词。

## 怎么改成你的产品

把默认输入换成你自己的产品：

- `product`：你的产品名，例如简历优化器、记账助手、课程笔记工具。
- `domain`：你的产品所在场景。
- `audience`：第一批真实用户，不要写“所有人”。
- `scene`：用户为什么会在手机上打开它。
- `material`：用户在移动端能快速输入的最小信息。
- `publishTarget`：先发 H5 链接、社群内测，还是准备小程序提审。

然后让 AI 帮你做一次压缩：

```text
请把我的 Web App 主路径压缩成移动端 H5 版本。
只保留用户在手机上 3 分钟内能完成的动作。
请说明保留什么、删除什么、为什么。
第一版不要接登录、支付、真实模型和小程序提审。
```

这就是本项目真正训练的能力：用 AI 把产品想法做成一个更小、更快、更容易验证的移动端版本。
