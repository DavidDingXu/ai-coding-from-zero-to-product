# 01 原始想法

`web-app` 已经能在电脑上生成、筛选和保存推荐。下一步不能只往里面加功能。

真实用户经常在手机上看到素材，比如文章、群聊、短视频标题或一个产品灵感。这个场景不适合长表单、10 条结果和复杂筛选。移动端版本要更短：输入少一点，结果少一点，动作更直接。

这个需求来自一个真实移动端场景：用户在地铁上、会议间隙或群聊里看到一条信息，只愿意花 3 分钟把它变成今天可以处理的一件事。此时他不会维护完整任务库，也不会筛 10 条结果。

本轮想法：

```text
把 AI 个人效率助手压缩成一个 H5 / 小程序形态的移动端工具。
用户在手机上输入产品、领域、读者和素材，得到 3 条今日推荐，收藏一条，并复制一段行动文本。
```

这个项目训练的不是 H5 语法，而是 AI 编程里的产品压缩能力：当你用 AI 做自己的产品时，能不能把完整工作台缩成一个手机上能完成的版本。

真实约束：

- 不把 Web App 的长表单和 10 条结果原样搬到手机端。
- 不接登录、openid、云数据库、订阅消息和真实小程序审核。
- 不要求用户在手机上填写 API Key 或长篇隐私内容。
- H5 本地验证和小程序提审必须分开表达。
