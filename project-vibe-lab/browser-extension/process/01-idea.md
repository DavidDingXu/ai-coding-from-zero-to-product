# 01 原始想法

「AI 个人效率助手」已经有网页应用和 H5 版本，但用户的素材通常来自正在浏览的网页。

这个项目做一个浏览器插件 MVP：用户在网页上选中一段内容，点击插件，把标题、链接和选中文本保存成网页笔记素材卡片。

这个需求来自一个真实使用场景：用户正在读文章、看竞品页面或整理案例，看到一段内容想带回自己的计划工具。如果让他复制、切窗口、打开 Web App、粘贴、再保存，动作太重。插件适合把“当前网页素材”这一步变轻。

第一批读者：

- 效率工具用户。
- 运营人员。
- 独立开发者。
- 想用 AI 做自己产品的人。

第一版只做当前页面采集，不做自动爬虫和远程同步。

真实约束：

- 必须由用户点击触发，不能后台自动采集。
- 只读取当前激活页面，不申请全站 host permissions。
- 不读取 Cookie、表单、私信、密码和登录态。
- 当前只保存本地素材卡片，不上传远程数据库。
- 当前不调用真实模型，不把本地摘要写成模型总结。
