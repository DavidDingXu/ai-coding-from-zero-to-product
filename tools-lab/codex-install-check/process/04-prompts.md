# 04 关键提示词

## 创建自检工具

```text
请在 tools-lab/codex-install-check 下创建一个零依赖的 Node.js 自检工具。

目标：
帮助 AI 编程初学者检查本机是否具备跟练 Codex 的基础环境。

要求：
1. 检查 node、npm、codex 命令是否可用。
2. 检查 Codex home，默认 ~/.codex，支持 CODEX_HOME。
3. 检查 config.toml、auth.json、全局 AGENTS.md。
4. 检查项目 AGENTS.md，必须从当前目录向上查找。
5. 输出文本报告和 JSON 报告。
6. 不读取密钥内容。
7. 使用 node:test，先写失败测试，再写实现。
```

## 为什么这样写

这个提示词没有说“帮我写一个安装教程”。

安装教程无法判断读者机器状态。自检工具可以。

这里最关键的要求是第 4 条：

```text
检查项目 AGENTS.md，必须从当前目录向上查找。
```

读者通常会在子模块里运行命令。如果只检查当前目录，会误报项目规则缺失。

## 不推荐的提示词

```text
帮我写一个 Codex 安装脚本。
```

这太危险，也不适合第 02 讲。

不同系统、账号、网络、安装方式差异很大。入门阶段更应该先做状态检查，而不是自动替用户改环境。
