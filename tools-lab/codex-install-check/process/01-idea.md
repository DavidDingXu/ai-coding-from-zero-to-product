# 01 原始想法

## 一句话

做一个 Codex 安装自检工具。

## 为什么需要

很多人第一次学 AI 编程，不是卡在代码，而是卡在环境。

常见问题包括：

- 终端找不到 `node`。
- 终端找不到 `npm`。
- 终端找不到 `codex`。
- 不知道 Codex 配置文件在哪里。
- 不知道自己有没有登录成功。
- 不知道 `AGENTS.md` 应该放在哪里。

如果文章只写安装步骤，读者照着做失败后，仍然不知道自己卡在哪一层。

所以第 02 讲需要一个自检工具，让读者先把环境状态看清楚。

## 第一版只做什么

- 检查 Node.js。
- 检查 npm。
- 检查 Codex CLI。
- 检查 Codex home。
- 检查 `config.toml`。
- 检查登录缓存。
- 检查全局和项目 `AGENTS.md`。

## 第一版不做什么

- 不安装 Codex。
- 不替代 `codex doctor`。
- 不读取密钥内容。
- 不判断模型额度。
- 不连接远程模型。

## 学习重点

安装不是抄命令，而是会判断环境状态。

读者要学会把“我这里不行”拆成：

```text
命令不存在
登录没完成
配置文件没准备
项目规则没加载
```
