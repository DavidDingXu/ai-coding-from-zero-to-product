# 08 复盘

## 做对的地方

第一，把安装问题拆成了检查项。

读者看到失败项后，能知道自己是缺 Node、缺 npm、缺 Codex CLI，还是缺项目说明文件。

第二，没有自动修改用户环境。

第 02 讲的目标是“看清楚状态”，不是写一个跨系统安装脚本。

第三，真实运行暴露了子目录误报问题，并用测试固定下来。

这是一个很适合写进文章的点：AI 编程不是一次生成就结束，而是要用真实运行结果反推工具设计是否合理。

## 还可以继续补什么

- 检查 `codex doctor --summary`。
- 检查 Git 是否存在。
- 检查当前目录是否在 Git 仓库内。
- 检查 shell completion 是否配置。
- 检查 `CODEX_HOME` 指向是否异常。

## 文章里要避免的说法

不要写：

```text
只要 7/7 通过，你的 Codex 就一定没问题。
```

应该写：

```text
这个自检只覆盖入门阶段最常见的本地环境问题。更完整的诊断仍然要看 Codex 官方的 codex doctor。
```
